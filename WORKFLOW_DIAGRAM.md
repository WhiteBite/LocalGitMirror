## Пошаговый workflow

### День 1: Первый запуск (Понедельник)

```
РАБОЧИЙ ПК                          ДОМАШНИЙ СЕРВЕР
─────────────────────────────────────────────────────────

1. Скачать backup_work_stealth.bat
   ├─ Dashboard → Data Sync Procedures
   └─ Кнопка "Download Stealth Script"
                                    ↓
2. Запустить скрипт
   ├─ backup_work_stealth.bat dandan
   ├─ git bundle create (полный)
   ├─ 7z encrypt (AES-256)
   └─ dump_MyProject_20240520_1800.dmp (50 MB)
                                    ↓
3. Загрузить через браузер
   ├─ https://192.168.1.100:443/files
   ├─ File Manager → work-sync
   └─ Upload dump_MyProject_*.dmp
                                    ↓
4. Синхронизировать
                                    ├─ Dashboard → Data Sync Procedures
                                    ├─ Кнопка "Sync Changes to Workspace"
                                    ├─ Случайная задержка 2-4 сек (OpSec)
                                    ├─ 7z x -p[password] (распаковка)
                                    ├─ git fetch + reset --hard
                                    └─ ✅ Готово!
```

### День 2: Инкрементальный бэкап (Вторник)

```
РАБОЧИЙ ПК                          ДОМАШНИЙ СЕРВЕР
─────────────────────────────────────────────────────────

1. Работаете весь день
   ├─ Редактируете файлы
   ├─ Коммитите изменения
   └─ git commit -m "Tuesday work"
                                    ↓
2. Конец дня - запустить скрипт
   ├─ backup_work_stealth.bat dandan
   ├─ Проверка .last_sync (есть!)
   ├─ git bundle create (ДЕЛЬТА!)
   │  └─ Только коммиты после понедельника
   ├─ 7z encrypt
   └─ dump_MyProject_20240521_1800.dmp (8 MB!)
                                    ↓
3. Загрузить
   ├─ https://192.168.1.100:443/files
   └─ Upload dump_MyProject_*.dmp
                                    ↓
4. Синхронизировать
                                    ├─ Проверка конфликтов
                                    │  └─ git status --porcelain
                                    ├─ Проверка PROJECT_NAME
                                    │  └─ dump_MyProject_* == current_repo
                                    ├─ Случайная задержка (OpSec)
                                    ├─ Распаковка
                                    ├─ git fetch + reset --hard
                                    └─ ✅ Готово!
```

## Состояния и переходы

```
┌─────────────────────────────────────────────────────────────┐
│                    СОСТОЯНИЯ СИНХРОНИЗАЦИИ                  │
└─────────────────────────────────────────────────────────────┘

IDLE (Ожидание)
    ↓
    ├─ Пользователь нажимает "Sync"
    ↓
PREPARING (Подготовка)
    ├─ Случайная задержка 3-5 сек
    ├─ Показывает countdown: "Подготовка... (3с)"
    ↓
SYNCING (Синхронизация)
    ├─ Проверка конфликтов
    ├─ Проверка PROJECT_NAME
    ├─ Случайная задержка 2-4 сек (backend)
    ├─ Распаковка дампа
    ├─ git fetch + reset --hard
    ↓
SUCCESS или ERROR
    ├─ Показывает результат
    ├─ Логирует в System Log
    ↓
IDLE (Готово к следующей синхронизации)
```

## Обработка ошибок

```
┌─────────────────────────────────────────────────────────────┐
│                    ОБРАБОТКА ОШИБОК                         │
└─────────────────────────────────────────────────────────────┘

Ошибка: Uncommitted changes detected
├─ Причина: На домашнем ПК есть незакоммиченные изменения
├─ Решение:
│  ├─ git add .
│  ├─ git commit -m "Home changes"
│  └─ Повторить синхронизацию
└─ Логирование: System Log

Ошибка: Dump is for 'ProjectX' but current repo is 'ProjectY'
├─ Причина: Загружен дамп для другого проекта
├─ Решение:
│  ├─ Выбрать правильный проект в Dashboard
│  └─ Загрузить правильный дамп
└─ Логирование: System Log

Ошибка: Decryption failed - wrong password
├─ Причина: Неверный пароль
├─ Решение:
│  ├─ Проверить SYNC_PASSWORD в .env
│  └─ Использовать правильный пароль
└─ Логирование: System Log

Ошибка: Bundle not found after decryption
├─ Причина: Поврежденный дамп или неверный пароль
├─ Решение:
│  ├─ Создать новый дамп на работе
│  └─ Загрузить снова
└─ Логирование: System Log
```

## OpSec (Скрытность)

```
┌─────────────────────────────────────────────────────────────┐
│                    OpSec МЕХАНИЗМЫ                          │
└─────────────────────────────────────────────────────────────┘

1. Имя файла
   ├─ Выглядит как: dump_ProjectName_20240520_1800.dmp
   ├─ Похоже на: Windows memory dump
   └─ NDR видит: Обычный файл логов

2. Расширение
   ├─ .dmp (не .zip, не .tar, не .bundle)
   └─ Маскирует истинное содержимое

3. Шифрование
   ├─ AES-256 с hidden headers (-mhe=on)
   ├─ Невозможно определить содержимое
   └─ Требует пароль для распаковки

4. Случайные задержки
   ├─ Frontend: 3-5 сек перед отправкой
   ├─ Backend: 2-4 сек перед распаковкой
   └─ Разрывает корреляцию "upload → process"

5. Скучный заголовок
   ├─ X-Session-ID (вместо X-API-Key)
   └─ Выглядит как обычный session token

6. Минимальное логирование
   ├─ SILENT_GIT=true
   ├─ Скрывает Git операции
   └─ Выглядит как обычная работа

7. Автоматическая ротация
   ├─ dump_ProjectName_20240520_1800.dmp
   ├─ dump_ProjectName_20240521_1800.dmp
   └─ Выглядит как регулярные логи
```

## Безопасность данных

```
┌─────────────────────────────────────────────────────────────┐
│                    БЕЗОПАСНОСТЬ                             │
└─────────────────────────────────────────────────────────────┘

Шифрование:
├─ AES-256 (военный стандарт)
├─ Hidden headers (-mhe=on)
└─ Требует пароль для распаковки

Целостность:
├─ Git bundle (криптографически подписан)
├─ Проверка PROJECT_NAME
└─ Проверка конфликтов перед применением

Конфиденциальность:
├─ HTTPS (TLS 1.3)
├─ Self-signed сертификат
└─ Локальная сеть (не интернет)

Отсутствие следов:
├─ Временные файлы удаляются сразу
├─ Логи минимальны (SILENT_GIT)
└─ Выглядит как обычная работа
```
