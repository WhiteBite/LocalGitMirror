# ✅ Stealth Sync - Чеклист установки

## 🏠 Домашний сервер (один раз)

### Предварительные требования
- [ ] Windows 10/11 или Linux
- [ ] Python 3.12+
- [ ] Node.js 18+
- [ ] Git установлен
- [ ] 7-Zip установлен (`C:\Program Files\7-Zip\7z.exe`)

### Установка зависимостей

```cmd
# Backend
cd backend
pip install -r requirements.txt
# ✅ Проверьте что установилось без ошибок

# Frontend
cd ../frontend
npm install
npm run build
# ✅ Проверьте что dist/ создана
```

### Конфигурация

- [ ] Отредактируйте `.env`:
  ```env
  WEB_PORT=443
  GIT_PORT=8444
  STORAGE_PATH=E:/kryptonit
  API_KEY=stealth-bridge-token-2026
  SILENT_GIT=true
  SYNC_PASSWORD=dandan
  ```

- [ ] Проверьте что 7-Zip установлен:
  ```cmd
  "C:\Program Files\7-Zip\7z.exe" --version
  # ✅ Должна показать версию
  ```

- [ ] Узнайте домашний IP:
  ```cmd
  ipconfig | findstr "IPv4"
  # ✅ Запишите адрес вида 192.168.x.x
  ```

### Запуск сервера

- [ ] Запустите backend:
  ```cmd
  python backend/run.py
  # ✅ Должно показать: "Uvicorn running on https://0.0.0.0:443"
  ```

- [ ] Откройте в браузере:
  ```
  https://192.168.1.100:443
  # ✅ Должна открыться Dashboard
  # (игнорируйте предупреждение о сертификате)
  ```

- [ ] Проверьте что работает:
  - [ ] Dashboard загружается
  - [ ] File Manager доступен
  - [ ] System Log показывает логи
  - [ ] Data Sync Procedures видна

### Первый тест

- [ ] Откройте Dashboard → Data Sync Procedures
- [ ] Скачайте `backup_work_stealth.bat`
- [ ] Проверьте что файл скачался

## 💼 Рабочий ПК (каждый день)

### Предварительные требования

- [ ] Git установлен
- [ ] 7-Zip установлен
- [ ] Браузер (Chrome/Firefox)
- [ ] Доступ к домашнему серверу (192.168.1.100:443)

### Первый запуск

- [ ] Скачайте `backup_work_stealth.bat` с домашнего сервера
- [ ] Поместите в корень проекта:
  ```
  C:\Users\YourName\Projects\MyProject\backup_work_stealth.bat
  ```

- [ ] Проверьте что вы в Git репозитории:
  ```cmd
  cd C:\Users\YourName\Projects\MyProject
  git status
  # ✅ Должно показать: "On branch main"
  ```

- [ ] Создайте первый дамп:
  ```cmd
  backup_work_stealth.bat dandan
  # ✅ Должно показать:
  # [*] Generating memory dump...
  # [+] SUCCESS: Memory dump generated
  # File: dump_MyProject_20240520_1800.dmp (50 MB)
  ```

- [ ] Проверьте что файл создан:
  ```cmd
  dir dump_*.dmp
  # ✅ Должен быть dump_MyProject_*.dmp
  ```

### Загрузка на сервер

- [ ] Откройте браузер: `https://192.168.1.100:443`
- [ ] Перейдите: Dashboard → File Manager
- [ ] Выберите папку: Shared Folders → work-sync
- [ ] Загрузите файл: `dump_MyProject_*.dmp`
- [ ] Ждите завершения загрузки
- [ ] ✅ Файл должен появиться в списке

### Синхронизация

- [ ] Откройте Dashboard → Data Sync Procedures
- [ ] Нажмите кнопку: "Sync Changes to Workspace"
- [ ] Ждите сообщения: "Sync applied successfully"
- [ ] ✅ Синхронизация завершена!

### Проверка результата

- [ ] Откройте File Manager
- [ ] Проверьте что файлы обновились
- [ ] Откройте System Log
- [ ] Проверьте что операция залогирована

## 🆘 Проверка если не работает

### Сервер не запускается

- [ ] Проверьте Python:
  ```cmd
  python --version
  # ✅ Должно быть 3.12+
  ```

- [ ] Проверьте зависимости:
  ```cmd
  cd backend
  pip install -r requirements.txt
  ```

- [ ] Проверьте порт 443:
  ```cmd
  netstat -ano | findstr :443
  # Если занят - измените WEB_PORT в .env
  ```

### Дамп не создается

- [ ] Проверьте 7-Zip:
  ```cmd
  "C:\Program Files\7-Zip\7z.exe" --version
  # ✅ Должна показать версию
  ```

- [ ] Проверьте что вы в Git репозитории:
  ```cmd
  git status
  # ✅ Должно быть "On branch main"
  ```

- [ ] Проверьте пароль:
  ```cmd
  backup_work_stealth.bat dandan
  # Используйте пароль из .env SYNC_PASSWORD
  ```

### Синхронизация не работает

- [ ] Проверьте что дамп загружен:
  ```
  Dashboard → File Manager → work-sync
  # ✅ Должен быть dump_ProjectName_*.dmp
  ```

- [ ] Проверьте пароль:
  ```
  .env: SYNC_PASSWORD=dandan
  ```

- [ ] Проверьте логи:
  ```
  Dashboard → System Log
  # ✅ Должны быть записи о синхронизации
  ```

- [ ] Проверьте что нет конфликтов:
  ```cmd
  git status
  # ✅ Должно быть "nothing to commit"
  ```

## 🔒 Безопасность

### Обязательно сделайте

- [ ] Измените пароль в `.env`:
  ```env
  SYNC_PASSWORD=ваш_надежный_пароль_32_символа
  ```

- [ ] Измените API ключ в `.env`:
  ```env
  API_KEY=ваш-уникальный-ключ-2026
  ```

- [ ] Проверьте что 7-Zip установлен правильно
- [ ] Не открывайте порт 443 в роутере без необходимости
- [ ] Используйте VPN для доступа из интернета

### Рекомендуется

- [ ] Регулярно проверяйте System Log
- [ ] Используйте сильные пароли (32+ символа)
- [ ] Не делитесь паролем с кем-либо
- [ ] Регулярно обновляйте 7-Zip
